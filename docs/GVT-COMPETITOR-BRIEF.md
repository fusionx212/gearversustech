# GVT Competitor Brief — Comparison Engine Positioning

**Date:** 2026-07-16  
**Site:** https://gearversustech.com  
**Locked positioning:** UK **comparison** site (X vs Y, best under £Y). Discovery via SEO / AEO / GEO. Convert via buy boxes, kit CTA, email. Hubs secondary — not a desk/room lifestyle brand.

Research method: public page fetches + pattern notes (Versus.com CloudFront-blocked; patterns from category indexes + secondary sources).

---

## 1. RTINGS.com

**What they are:** Structured lab tests, usage scores, tools (compare / table / graphs). Authority via methodology depth.

### Steal-worthy
1. **Usage / intent entry** — category landing opens into Best-by-usage and Best-by-price slices (Gaming, Budget, OLED…), not a blog feed.
2. **Score-first compare tool** — side-by-side metrics before narrative; winner per dimension is scannable.
3. **Persistent tools nav** — Compare / Results Table / Review Index as first-class IA; content is a graph, not posts.
4. **Trust via process volume** — “N products bought and tested” + pipeline transparency.
5. **Dense internal linking** — every review fans out to related models and “best for X” money pages.

### Avoid
1. **Lab-tool overwhelm** — custom ratings / graph tools scare casual UK shoppers; GVT should stay verdict-readable in 10 seconds.
2. **US-default commerce** — prices/retailers not UK-native; our edge is live Amazon.co.uk / eBay + fail-soft.

---

## 2. NYT Wirecutter

**What they are:** Verdict-first editorial picks with clear hierarchy (Top / Budget / Also consider).

### Steal-worthy
1. **Pick hierarchy above the fold** — “Everything we recommend” with Top pick / Budget pick cards before the essay.
2. **One-line why + honest caveat** — each pick states strength *and* who should skip (price, fit, ear size…).
3. **Multi-retailer buy options** early — conversion without burying the verdict.
4. **Who this is for / How we tested** as trust modules, not buried footnotes.
5. **Updated date + author accountability** — AEO-friendly freshness signals.

### Avoid
1. **Essay length before decision** — long lifestyle openers delay the buy; GVT searchers already have intent.
2. **White “newsletter column” shell** — competent but generic; GVT keeps workshop dark/amber system.

---

## 3. Versus.com (vs engines)

**What they are:** Spec-database A vs B / rankings with filters and a proprietary score.

### Steal-worthy
1. **A vs B as the product** — URLs and UI scream comparison, not articles.
2. **Filter chips by job** — Budget, battery, screen size → intent routes into the graph.
3. **Lead/trail per attribute** — winner marks on each row of the spec table.
4. **Endless related “X vs Y” links** — SEO cluster machine from every product page.

### Avoid
1. **Spec soup without a human verdict** — numbers without “buy this if…” convert poorly for mid-ticket gear.
2. **Popularity-weighted scores** — can feel gamed; GVT sticks to weighted methodology (perf/value/build/ease).

---

## 4. Tom's Guide / TechRadar (UK-relevant roundups)

**What they are:** Jump-to TOC “best of” articles with Best overall / Best budget / Best under £X / FAQs / How we test.

### Steal-worthy
1. **Jump links** — Best overall · Best under £250 · Best under £100 (price bands as H2 anchors).
2. **Explicit “#1 pick to buy today”** — single decisive answer for impatient readers / AI overviews.
3. **FAQ blocks** — AEO: commuting, calls, studying, glossary.
4. **Budget bands in GBP** — TechRadar often dual-prices; GVT should be GBP-first always.
5. **Test scores / latest updates** sections for freshness.

### Avoid
1. **Affiliate link farms + sidebar clutter** — dilutes trust; keep CTAs in buy boxes + sticky bar only.
2. **Thin “also tested” lists** — name-drops without scores; every GVT row needs a rating or a reason omitted.

---

## 5. Garage Gym Reviews

**What they are:** Category-authority money pages; homepage = roundup index by equipment type.

### Steal-worthy
1. **Category as IA** — Strength / Cardio / Recovery clusters, each linking to “Best X” money pages.
2. **How we test** as a first-class section with per-product methodology deep links.
3. **Budget sub-cluster** — Best under $500 / Best budget treadmills as sibling money pages.
4. **Team / credentials** for trust (UK equivalent: methodology + disclosure, not fake bylines).
5. **Deals strip** as conversion adjacency without owning the homepage.

### Avoid
1. **Homepage as endless card grid** — exhausting; GVT uses featured compare + clustered lists.
2. **Room-build lifestyle framing as primary** — useful secondary (our `/spaces/`), never the entry promise.

---

## 6. Which? (UK trust patterns)

**What they are:** Lab-tested UK consumer authority; Best Buy / Don't Buy; paywall on full scores.

### Steal-worthy
1. **Pros / cons on every listing** — scannable honesty.
2. **Independence messaging** — editorial not sold; GVT mirrors via disclosure + “rankings never sold”.
3. **Retailer price aggregation mindset** — UK prices front and centre.
4. **Don't-buy framing** — negative recommendations build trust; GVT “who should not buy” already aligns.

### Avoid
1. **Hard paywall on the verdict** — kills SEO/AEO and affiliate conversion; GVT stays free + affiliate.
2. **Filter UI without content depth** — tool without narrative; we need both score and short verdict.

---

## 7. Niche “A vs B” affiliate independents (keyboard/mouse blogs)

**Patterns observed across strong independents (e.g. RTINGS-adjacent niche sites, desk-setup vs posts):**

### Steal-worthy
1. **Title = query** — `Keychron Q1 vs Keychron V1` matches search exactly.
2. **Spec table mid-page + buy CTAs above and below**.
3. **Related compares footer** — same brand family / same budget band.
4. **Email after decision** — capture once the verdict is consumed.

### Avoid
1. **Above-the-fold banner ads before the winner**.
2. **Outdated prices with no “checked” date** — erodes trust; fail-soft “Price updating” beats a lie.

---

## GVT should adopt / GVT should beat

| Pattern | Adopt from | GVT should beat by… |
|---|---|---|
| Verdict / pick hierarchy first | Wirecutter, Tom's Guide | Show Our pick + score + who-should-skip **before** long HTML; sticky UK buy bar |
| Score clarity on A vs B | RTINGS, Versus | Weighted /10 with fail-soft UK £ from Creators API — not US MSRP |
| Category → cluster → spoke IA | Garage Gym Reviews, RTINGS | Subcategory clusters on indexes; hubs stay secondary |
| Jump-to / FAQ for AEO | TechRadar, Tom's Guide | FAQPage schema + details/summary on every compare |
| Related X vs Y graph | Versus, niche blogs | Same-subcategory related compares (not only “all gaming”) |
| Trust / independence | Which?, Wirecutter | Methodology lite on home + disclosure near CTAs; never paywall |
| Conversion placement | Wirecutter buy options | Amazon + eBay dual CTAs, kit CTA, email — no clutter sidebars |
| Budget / under-£Y entry | TechRadar, GGR | `/best/` + homepage intent chips for under-£ compares |
| Avoid lifestyle-first home | (anti-pattern from GGR room framing / old GVT hubs) | Comparison-engine homepage; `/spaces/` quiet link only |

---

## Implementation priority (this ship)

1. Homepage IA = categories → featured compare → latest → how we pick → kit/email (spaces quiet).
2. Compare template = verdict bar → buy boxes → body → sticky buy → FAQ + schema → related cluster → kit/email.
3. Category indexes = subcategory clusters so the graph “fits together”.
4. Keep existing schema, affiliate rails, fail-soft prices, design tokens (graphite/amber).
